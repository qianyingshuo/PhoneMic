# PhoneMic 使用说明书

## 1. 产品是什么

PhoneMic 解决电脑上语音输入不方便的问题。它通过局域网把手机和电脑连起来，你用手机当话筒，借助手机上的语音识别（比如豆包输入法），说出的话会实时转成文字，然后自动填到电脑当前光标所在的位置。

**核心功能：**

- **手机扫码连接：** 电脑上显示二维码，手机扫一下就能通过浏览器连上，不用安装任何 App。
- **实时预览：** 你用手机语音输入时，电脑屏幕下方会弹出一个半透明小窗口，能看到你正在说什么。
- **一键上屏：** 手机发送后，文字会自动粘贴到电脑当前激活的输入框里。手机端可以选“自动发送”或“手动发送”。
- **语音命令：** 可以自己定义规则，比如你说“确定”，电脑就自动按回车；你说“打开计算器”，电脑就运行计算器程序。这样能省去很多手动操作。
- **局域网安全：** 所有数据只在你自己的局域网里传输。请注意，传输是明文的（没有额外加密），所以请只在您信任的网络下使用，比如家里的 WiFi 或公司的内部网络，不要在公共咖啡厅、机场等不可信的 WiFi 下使用。

## 2. 电脑端怎么用

### 2.1 启动和选择 IP

1. **启动程序：** 双击 PhoneMic 的图标。
2. **选择 IP 地址：**
   - 如果电脑只有一个局域网 IP，程序会自动选好并启动。
   - 如果有多个 IP（比如同时连着有线和无线），会弹出一个窗口让你选。选一个手机能访问到的 IP 就行，一般是手机连接的 WiFi 对应的那个。
   - **注意：** 手机和电脑必须在同一个局域网（同一个路由器或热点）下才能用。
3. **主界面：** 选好 IP 后，程序会显示一个二维码。

### 2.2 连接手机

1. **扫二维码：** 用手机自带的相机或者微信、支付宝的扫码功能，扫电脑屏幕上的二维码。
2. **浏览器打开：** 手机浏览器会自动打开一个页面，那就是手机端的输入界面。
3. **连上了：** 手机页面显示“已连接”，电脑主界面和右下角托盘图标也显示“已连接”，就说明连好了。

### 2.3 悬浮窗

- **自动出现：** 当你在手机端输入文字时，电脑屏幕底部中央会弹出一个半透明小窗，实时显示你输入的内容。
- **自动消失：** 如果你停止输入超过 **3 秒**，或者手机端清空了输入框，悬浮窗就会自己隐藏。
- **不干扰你：** 悬浮窗不会抢走当前电脑程序的焦点，你可以继续做别的事。

### 2.4 系统托盘（右下角图标）

PhoneMic 运行时，电脑任务栏右下角会显示一个小图标。

- **状态提示：** 红点表示没连上，绿点表示已连接。
- **右键菜单：** 鼠标右键点击图标，你可以：
  - **显示主界面：** 重新打开带二维码的窗口。
  - **命令配置：** 打开语音命令管理窗口（详见第4章）。
  - **退出：** 关闭程序。
- **左键点击：** 可以快速显示或隐藏主界面。

## 3. 手机端怎么用

### 3.1 页面长什么样

打开手机浏览器里的 PhoneMic 页面，你会看到一个简洁的聊天风格界面：

- **顶部：** 显示连接状态，比如“已连接”或“正在重连...”。
- **中间：** 显示你发送到电脑的文字记录，每条是一个气泡（最多保留50条）。
- **底部：** 一个输入框，左边“清空”按钮，右边“发送/自动”按钮。

### 3.2 输入和预览

1. **自动聚焦：** 页面一打开，输入框会自动获得焦点，同时弹出手机输入法。
2. **实时预览：** 输入文字时，电脑上的悬浮窗会同步显示。
3. **自动换行：** 输入框会根据文字长度自动调整高度。

### 3.3 自动发送模式（默认）

默认就是自动模式，特别适合语音输入：

- **怎么判断语音结束：** 当你在手机输入法（比如豆包）里语音输入时，识别中的文字下面会有下划线。识别完成、下划线消失后，PhoneMic 会自动把文字发送到电脑。
- **发完之后：** 发送成功后，输入框自动清空并保持聚焦，你可以直接开始下一轮语音输入，不用点任何东西。
- **模式标志：** 右侧按钮显示 **“自动”**，底色是橙色。

### 3.4 手动发送模式

如果你想自己控制发送时机（比如打字输入），可以切到手动模式：

- **怎么切换：** 当输入框为空时，输入框内部右侧会出现一个闪电符号（⚡），点它就能在自动和手动之间切换。
- **手动模式的样子：**
  - 右侧按钮显示 **“发送”**，底色是绿色。
  - 语音识别完成后不会自动发送，你需要点“发送”按钮或按回车键。
- **切换时有反馈：** 手机可能会轻微震动一下（看你的设备是否支持），按钮文字和颜色会立刻变化。

### 3.5 发送文字（手动模式）

- **点“发送”：** 把输入框里的文字发到电脑。
- **按回车：** 在输入框里按手机输入法的回车键，一样可以发送。
- **上屏效果：** 发送成功后，文字会立刻粘贴到电脑当前光标所在的地方。
- **记录和清空：** 发送的文字会显示在聊天记录里，同时输入框自动清空并保持聚焦。

### 3.6 清空输入框

- **点“清空”：** 输入框左侧的按钮，一点就清空所有内容。电脑上的悬浮窗也会跟着消失。

### 3.7 重新发历史消息

1. **长按气泡：** 在聊天记录区，长按某一条你之前发过的文字。
2. **自动填充：** 长按后，那条消息的内容会加到输入框末尾，你可以改完再发。
3. **短按重发：** 直接短按气泡，会弹出一个确认框问你要不要重发。确认后就直接发到电脑了。

## 4. 语音命令（扩展功能）

你可以定义一些 **语音命令**：当手机发送的文本匹配了你设好的规则时，就不再只是粘贴文字，而是执行**模拟按键**或**运行外部程序**（比如打开计算器、运行脚本、批量删除等）。

### 4.1 打开命令管理窗口

有两种方式打开：

- **主界面菜单：** 点主窗口顶部的 **“工具”** → **“命令配置...”**。
- **系统托盘右键：** 右键点右下角托盘图标 → **“命令配置...”**。

命令管理窗口是一个单独的对话框，你可以在这里添加、修改、删除、启用/禁用命令，还能调整顺序。

### 4.2 新增一条命令

点 **“新增”**，填下面这些信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 让你自己明白这条命令是干啥的 | “清空输入框” |
| 匹配类型 | `完全匹配` 或 `前缀匹配` | 完全匹配 |
| 匹配模式 | 触发命令的文字 | `清空` |
| 动作类型 | `模拟按键` 或 `运行程序` | 模拟按键 |
| 动作参数 | 按键序列 或 命令行 | `ctrl+a, delete` |

保存后命令立刻生效，不用重启程序。

### 4.3 匹配规则

- **完全匹配**：手机发的文字必须和“匹配模式”**一模一样**才触发。
  - 例子：匹配模式 `确定` → 发“确定”会触发，发“确定了”不会。
- **前缀匹配**：手机发的文字以“匹配模式”**开头**就触发。
  - 例子：匹配模式 `请帮我记录 ` → 发“请帮我记录 今天开会”会触发，并且 `今天开会` 会作为 `{content}` 传进去。

### 4.4 动作类型

#### （1）模拟按键

在“动作参数”里输入**按键序列**，多个组合用逗号隔开，程序会按顺序一个个执行。每个组合里用 `+` 连接多个键。

**格式示例：**
- 单个键：`enter`、`tab`、`space`、`backspace`、`delete`、`up`、`down`
- 组合键：`ctrl+c`、`shift+alt+a`、`ctrl+shift+esc`
- 序列（先全选再删）：`ctrl+a, delete`
- 序列（复制再粘贴）：`ctrl+c, ctrl+v`

> 注意：按键名要用小写英文，多个键用 `+` 连，各组合之间用 **英文逗号** 隔开（前后可以有空格）。最多支持10个组合。

**常用命令例子：**

| 你想实现的 | 匹配模式 | 动作参数（模拟按键） |
|------------|----------|----------------------|
| 说“确定”就按回车 | `确定` | `enter` |
| 说“清空”就全选再删 | `清空` | `ctrl+a, delete` |
| 说“超级确定”就按 Ctrl+Enter（很多软件用这个组合来最终确定输入） | `超级确定` | `ctrl+enter` |

#### （2）运行程序

在“动作参数”里输入命令行，比如：

- `calc` → 打开计算器
- `notepad.exe D:\log.txt` → 用记事本打开某个文件
- `python D:\my_script.py {content}` → 把动态内容传给 Python 脚本

**安全性说明**：程序会把你的命令行**安全拆分**后再执行，不会经过系统 shell，能防止常见的命令注入。但如果你自己写了 `python -c "import os; os.system('{content}')"` 这种模板，里面的 `{content}` 还是可能被当作代码执行，请确认手机发送的内容是可信的（毕竟口头说出代码也很少见）。

**更多实用例子：**

| 你想实现的 | 匹配类型 | 匹配模式 | 动作参数（运行程序） |
|------------|----------|----------|----------------------|
| 把待办事项记到文本文件里 | 前缀匹配 | `请记录待办 ` | `cmd /c echo {content} >> D:\note\todo.txt` |
| 用 AutoHotkey 执行一个固定的自动化任务 | 完全匹配 | `帮我收集今天股票涨停信息` | `"C:\Program Files\AutoHotkey\AutoHotkey.exe" D:\my_scripts\collect_stocks.ahk` |
| 调用自己的智能助手脚本（带参数） | 前缀匹配 | `助手 ` | `python D:\my_agent.py --message "{content}"` |

> 解释：第一个例子，你说“请记录待办 明天去超市”，`明天去超市` 会被追加到 `D:\note\todo.txt` 文件里。第二个例子是一个完全匹配的命令，只要你说“帮我收集今天股票涨停信息”，就会直接运行你事先写好的 AutoHotkey 脚本（比如自动打开网页、抓取数据），不需要再传额外内容。第三个例子是唤醒你自己的 agent 脚本，可以去做更复杂的事，比如查天气、记笔记、控制设备等。

### 4.5 占位符（动态内容）

在“动作参数”里可以用这些占位符，程序会自动替换：

| 占位符 | 说明 | 示例（前缀匹配 `请帮我记录 `） |
|--------|------|-------------------------------|
| `{content}` | 匹配模式后面的部分 | 发“请帮我记录 明天开会”→ `明天开会` |
| `{prefix}` | 匹配模式本身 | `请帮我记录 ` |
| `{all_text}` | 手机发的完整文本 | `请帮我记录 明天开会` |

例子：命令参数 `python my_agent.py --msg "{content}"`，手机发“请帮我记录 测试”，实际执行 `python my_agent.py --msg "测试"`。

### 4.6 指定工作目录（{cwd:...}）

运行程序时，默认工作目录是 `%LOCALAPPDATA%\PhoneMic\exec_workdir`（比如 `C:\Users\你的用户名\AppData\Local\PhoneMic\exec_workdir`）。如果你想换到别的目录，可以在**命令行最前面**加上 `{cwd:"绝对路径"}`（这个标记会被自动去掉，不会传给实际程序）。

**语法要求：**
- 用紧凑格式，**大括号里面、冒号前后不能有空格**。
- 路径最好用双引号包起来（防止路径里有空格）。
- 可以用正斜杠 `/` 或双反斜杠 `\\`，`~` 会展开为用户目录。

**例子：**
```
{cwd:"D:\\my_work"} python script.py
{cwd:"~/Desktop"} echo hello > out.txt
```
上面第一条命令会在 `D:\my_work` 目录下执行 `python script.py`，第二条会在桌面目录执行。

> 提示：不需要自定义工作目录的话，直接写命令就行。默认目录会自动创建，你可以在那里放脚本或数据文件，方便管理。

### 4.7 管理命令列表

命令管理窗口里是一个表格，你可以：

- **启用/禁用**：勾选“启用”复选框，临时关闭某条命令。
- **优先级**：命令按**从上到下**的顺序匹配，命中第一条就停了。用 **“上移”/“下移”** 按钮调整顺序。
- **编辑/删除**：**双击任意行**就能修改；点操作列的“✖”按钮删除。
- **新增**：点窗口底部的 **“新增”** 按钮。

所有改动都会实时保存。

**给进阶用户的提示：** 如果你熟悉 JSON 格式，也可以直接用文本编辑器修改配置文件。文件位置是：
```
C:\Users\你的电脑用户名\AppData\Local\PhoneMic\config\commands.json
```
修改前建议先备份。注意保持 JSON 格式正确（比如不要缺逗号、引号必须成对），否则程序可能读不出来。

### 4.8 注意事项

- 运行外部程序时，可能会短暂弹出窗口（取决于程序自己）。
- 如果命令没触发，检查：
  - 命令是否**启用**了。
  - 匹配类型和匹配模式对不对（注意中文标点）。
  - 优先级顺序有没有问题（比如一个更宽泛的前缀命令把精确的命令挡了）。
- 按键序列里各组合之间会有 0.05 秒的延迟，保证操作能被正常响应。如果某个组合执行失败，后面的就不会再执行了。

## 5. 出问题了怎么办

- **连不上 / 页面空白：**
  - 检查手机和电脑是不是在同一个局域网里。
  - 确认电脑上的 PhoneMic 程序在运行。
  - 检查电脑防火墙有没有阻止 PhoneMic 使用它的端口（端口号看主界面，一般是 12000）。
  - 在手机浏览器里手动输入电脑主界面显示的完整地址，比如 `http://192.168.1.100:12000`。
- **文字上不了屏：**
  - 确认电脑上当前有可以输入文字的程序（比如记事本、聊天窗口、Word）并且处于激活状态。
  - 检查电脑安全软件是不是拦截了 PhoneMic 的模拟按键。
- **悬浮窗不显示：**
  - 确认手机已连接，而且你正在输入。
  - 检查显卡驱动，或者重启电脑试试。
- **程序闪退：**
  - 尝试以管理员身份运行 PhoneMic。
  - 如果经常闪退，可能是环境问题，可以联系开发者。
