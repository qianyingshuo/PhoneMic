# Quickstart & Verification Guide: key-mappings-redesign

本指南提供了在开发完成后，端到端手动验证双按键映射通道重构功能是否正常工作的步骤。

## 1. 启动 PhoneMic 服务

在工作区中运行以下命令来启动本软件的局域网服务端与 GUI 面板：

```bash
# 激活虚拟环境并启动
./.venv/bin/python -m phonemic.PhoneMic
```

---

## 2. 验证场景

### 场景 A：正常发送字词且不追加按键
1. 用手机扫码或在手机浏览器中打开服务页面。
2. 将手机端的 **⚡ 发送时追加** 下拉框设为 `"无 (不追加)"`。
3. 在手机端输入框输入文字 `测试不追加`，点击『发送』按钮或敲击输入法键盘上的回车。
4. **预期结果**：
   - 电脑端当前光标处成功录入文字 `测试不追加`。
   - 电脑端没有模拟追加任何按键动作（光标依然留在词尾，没有发生换行或跳格）。
   - 手机端聊天历史中追加一行绿色气泡：`测试不追加`。

### 场景 B：正常发送字词且自动追加回车
1. 将手机端的 **⚡ 发送时追加** 下拉框切换为 `"回车 (enter)"`。
2. 在手机端输入框输入文字 `测试追加回车`，点击『发送』按钮。
3. **预期结果**：
   - 电脑端当前光标处成功录入文字 `测试追加回车`，并且立即执行了一次换行动作（光标跳到了下一行行首）。
   - 手机端聊天历史中追加一行绿色气泡：`测试追加回车`。

### 场景 C：空白状态下单独发送 Tab 键
1. 将手机端的 **🎯 单独发按键** 下拉框设为 `"Tab (tab)"`。
2. **确保输入框完全为空**（没有任何字词）。
3. 点击右下角绿色的『发送』按钮（或在输入法键盘上敲击回车）。
4. **预期结果**：
   - 电脑端当前光标向前跳跃了一格（模拟了 `Tab` 按键）。
   - 手机端聊天历史中新增一行淡蓝色的系统模拟回显：`⌨️ [单独发送按键] Tab (tab)`。

### 场景 D：空白状态下无按键设置（空发拦截）
1. 将手机端的 **🎯 单独发按键** 下拉框切回到 `"无 (不发送)"`。
2. **确保输入框完全为空**。
3. 点击右下角绿色的『发送』按钮。
4. **预期结果**：
   - 没有任何动作，网页和电脑端无任何反应。
   - 开发者控制台没有发出任何 WebSocket 消息（连接处于空闲拦截状态）。
