# Interface Contracts: WebSocket 交互协议

手机网页端通过与服务端的常驻 WebSocket 双向连接进行配置获取、状态同步和数据发送。

---

## 1. 服务端下发数据协议

### 1.1 初始化及配置广播 (`type: "key_mappings"`)
当手机端首次连接建立成功，或 PC 端修改了按键映射保存后，服务端主动下发当前的完整按键映射列表。

* **数据格式**: JSON
* **协议范例**:
```json
{
  "type": "key_mappings",
  "data": [
    {
      "id": "none",
      "name": "无 (不追加)",
      "keys": ""
    },
    {
      "id": "a90f7bdf-1b8f-4cb1-8fe7-fb8db2fa3200",
      "name": "回车 (Enter)",
      "keys": "enter"
    }
  ]
}
```

### 1.2 主动重载指令 (`type: "reload"`)
在出现严重配置错乱、网络恢复或特定重置场景时，服务端下发此指令，强制手机端刷新页面。

* **数据格式**: JSON
* **协议范例**:
```json
{
  "type": "reload"
}
```

### 1.3 警告/错误信息推送 (`type: "warning"`)
当网页端发送的数据包含失效/已删除的按键映射 ID 时，服务端向网页端推送错误回执。

* **数据格式**: JSON
* **协议范例**:
```json
{
  "type": "warning",
  "text": "按键映射已在电脑端被删除，已自动重置为“无 (不追加)”"
}
```

---

## 2. 网页端发送数据协议

### 2.1 发送文本与追加按键序列 (`type: "send"`)
当用户在手机端网页点击发送（打字发送或语音发送）时，将数据发送给服务端。

* **数据格式**: JSON
* **协议范例**:
```json
{
  "type": "send",
  "text": "这是一段测试文字",
  "key_sequence": "enter"
}
```
*(注：如果当前网页选择的是“无 (不追加)”，则 `key_sequence` 为空字符串 `""`。服务端会向前兼容以前不带 `key_sequence` 字段的纯 `{"type": "send", "text": "内容"}` 消息格式)*
