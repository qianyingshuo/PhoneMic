# Quickstart Validation Guide: 快捷按键映射功能验证指南

本指南供研发人员和测试人员在本地快速运行并端到端验证“按键映射追加与管理”功能是否工作正常。

---

## 1. 验证前置准备
1. 启动 PC 客户端程序：
   ```bash
   poetry run python -m phonemic.PhoneMic
   ```
2. 确保在主界面选择正确的 IP 地址启动后端服务，电脑将弹出一个 Dashboard 窗口并展示二维码及服务链接。
3. 手机与电脑连入同一局域网（或使用本地测试：在电脑浏览器直接访问 Dashboard 页面下显示的 `http://<IP>:<Port>` 链接模拟手机网页端交互）。

---

## 2. 验证场景一: PC 端按键映射管理
1. **测试步骤**:
   * 在 PC 端 Dashboard 菜单栏点击：**设置 -> 按键映射** (或 Settings -> Key Mappings)。
   * 在弹出的对话框中，点击 **新增** (Add) 按钮。
   * 输入映射名称（如：“全选并回车”），并在按键序列中输入 `ctrl+a, enter`。
   * 点击 **确定** (Ok) 保存。
2. **预期结果**:
   * 系统应成功保存，在表格中能看到新添加的这行。
   * 打开软件生成的 `settings.json`（保存在配置目录内），应能看到在 `"key_mappings"` 下生成了一条分配有全新 UUIDv4 标识的配置 JSON。
   * 如果此时输入错误的按键名（如 `enter123`），保存时必须弹出“未知键名”的安全警告拦截并拒绝保存。

---

## 3. 验证场景二: 网页端实时同步与 LocalStorage 缓存
1. **测试步骤**:
   * 使用浏览器访问 Dashboard 二维码下方的网页地址。
   * 在手机端网页上点击左上角的“☰”菜单按钮展开侧边栏（Drawer）。
   * 观察“默认追加按键映射”的下拉列表。
   * 选中刚刚在 PC 端新建的“全选并回车”项。
   * 刷新网页。
2. **预期结果**:
   * 侧边栏的下拉列表中应能够即时渲染出 PC 端预设的所有映射（含刚刚新建的“全选并回车”项）。
   * 在刷新网页后重新展开侧边栏，下拉列表当前选中项应**依旧是**“全选并回车”，说明 LocalStorage 本地偏好已缓存成功。

---

## 4. 验证场景三: 发送后模拟追加按键 (端到端核心链路)
1. **测试步骤**:
   * 在电脑上新建一个文本文件，在里面写上三行文字，并将输入光标停留在第一行末尾。
   * 手机端网页侧边栏选择：`回车 (Enter)`。
   * 手机端输入框输入：“发送测试一”，并点击 **发送** 按钮。
2. **预期结果**:
   * 电脑的文本文件在光标处应立即被粘贴上了“发送测试一”这几个字。
   * 随后光标应**自动换行**，停留在第二行的开头（表示 `enter` 按键序列被成功触发并模拟执行）。
   * 整个过程（从点击发送到换行完成）体验应无任何明显停顿或延迟。

---

## 5. 验证场景四: 异常与失效 ID 处理 (防错乱安全降级)
1. **测试步骤**:
   * 手机端网页保持连通并选中“全选并回车”。
   * 在 PC 客户端的管理界面中，选中该条目并点击 **删除**。
   * 在手机端点击发送文本：“发送测试二”。
2. **预期结果**:
   * 电脑光标处成功粘贴文字“发送测试二”，但**不触发任何多余按键**（安全降级生效，不发生按键错乱）。
   * 手机网页端自动重置其下拉菜单选中项为“无 (不追加)”，LocalStorage 中的无效旧 UUID 自动被清理并重置为 `"none"`。
