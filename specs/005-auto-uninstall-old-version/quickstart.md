# Quickstart Validation Guide: 安装时自动检测并提示卸载旧版本

本指南用于指导开发者如何在 Windows 开发机/测试机上，对修改后的 NSIS 安装程序（`.nsi` 脚本）进行本地构建和端到端场景验证。

## 1. 准备工作 (Prerequisites)

1. **操作系统**：Windows 10 / 11 物理机或虚拟机。
2. **构建工具**：安装 NSIS 编译器。请确保 `makensis` 命令已加入系统的环境变量 `PATH`。
3. **测试包构建**：
   - 在项目根目录下，确保已拉取最新依赖并完成了 Nuitka 编译：
     ```bash
     python build_nuitka.py
     ```
     这会在 `build/` 目录下生成打包所需的源文件。

---

## 2. 编译安装包 (Build Setup Installer)

在 Windows 终端中运行以下命令来编译生成安装包：

```cmd
makensis makesetup.nsi
```
编译成功后，将在 `dist/` 目录下生成安装程序，例如 `dist/PhoneMic_Setup_*.exe`。

---

## 3. 测试验证用例 (Manual Test Cases)

### 场景 1：全新安装（无旧版本）
1. **测试前置条件**：确保系统中未曾安装过 PhoneMic（注册表中无相关项）。
2. **执行步骤**：双击运行生成的 `PhoneMic_Setup_*.exe`。
3. **预期结果**：安装程序直接进入常规安装界面，不弹出任何旧版检测或进程检测警告，安装顺利完成。

### 场景 2：进程运行前置拦截
1. **测试前置条件**：启动已安装的 `PhoneMic` 软件，确保其正在后台运行（托盘或窗口可见）。
2. **执行步骤**：运行新版 `PhoneMic_Setup_*.exe`。
3. **预期结果**：
   - 一启动立刻弹出警告框：“检测到 PhoneMic 正在运行，请先关闭程序再进行安装。”
   - 点击“确定”后，安装程序立刻退出，未进行任何文件覆写或卸载操作。

### 场景 3：正常升级并卸载旧版本
1. **测试前置条件**：
   - 确保旧版 PhoneMic 处于关闭状态，且系统已成功安装旧版本。
   - 在软件里随意更改一项配置（如语言或 HUD 倒计时），确保 `%LOCALAPPDATA%\PhoneMic\config\settings.json` 中有非默认的用户设置。
2. **执行步骤**：运行新版 `PhoneMic_Setup_*.exe`。
3. **预期结果**：
   - 弹出询问框：“检测到系统已安装 PhoneMic 的旧版本。是否先自动卸载旧版本以进行干净的升级安装？”
   - 点击**“是”**。
   - 稍等数秒，旧版卸载进程在后台同步执行完毕，原安装目录（如 `C:\Program Files\PhoneMic`）下的所有旧文件（如旧的 .dll / .pyd / .exe）被清理干净。
   - 接着新版安装流程继续执行并顺利完成。
   - 启动新版 PhoneMic，验证之前更改的语言/HUD配置依然生效，未被误删。

### 场景 4：取消升级
1. **测试前置条件**：同场景 3。
2. **执行步骤**：运行新版 `PhoneMic_Setup_*.exe`。
3. **预期结果**：
   - 弹出卸载确认询问框时，点击**“否”**。
   - 弹出提示框：“升级安装已取消。”
   - 安装程序安全退出，旧版本及个人配置均未发生改变。

### 场景 5：卸载器丢失的极端容错
1. **测试前置条件**：安装旧版本后，手动进入 `C:\Program Files\PhoneMic` 目录并彻底删除 `uninst.exe` 文件。
2. **执行步骤**：运行新版 `PhoneMic_Setup_*.exe`，在弹出升级确认框时选择**“是”**。
3. **预期结果**：
   - 弹出严重错误框：“未找到旧版本的卸载程序（uninst.exe已丢失）。请先手动删除以下安装目录以清除旧版本，然后再重新运行安装：C:\Program Files\PhoneMic”。
   - 点击确定后，安装程序安全退出。
