# Feature Specification: Windows 窗口最小化与关闭至系统托盘

**Feature Branch**: `001-minimize-to-tray`

**Created**: 2026-06-28

**Status**: Draft

**Input**: User description: "实现主窗口最小化和关闭时隐藏到系统托盘，不占用任务栏和 Alt+Tab 列表，且托盘菜单中的退出动作能正常彻底退出程序"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 窗口最小化隐藏 (Priority: P1)

用户可以通过点击主窗口右上角的“最小化”按钮将程序窗口彻底隐藏，仅在系统托盘保留图标，从而保持任务栏及任务切换（Alt+Tab）界面的整洁。

**Why this priority**: 该功能是本特性的核心诉求，直接解决程序在任务栏占用空间、干扰日常任务切换的痛点。

**Independent Test**:
- **自动化单元测试（WSL Linux环境）**：运行 pytest 自动化测试。通过 `pytest-qt` (qtbot) 触发主窗口的状态变更事件（模拟最小化），断言窗口的隐藏状态为 `True`（即 `isHidden()` 返回 `True`），并断言窗口状态标志位被重置为 `Qt.WindowNoState`。
- **手动实机测试（Windows环境）**：启动程序，点击“最小化”按钮，确认程序在任务栏和 Alt+Tab 中彻底消失，而系统托盘图标仍正常运行。双击系统托盘图标，窗口能够正常弹回并恢复在最前端显示。

**Acceptance Scenarios**:

1. **Given** 程序主窗口处于显示状态，**When** 用户点击窗口右上角的“最小化”按钮（或系统分发窗口最小化事件），**Then** 主窗口应当从桌面隐藏，且在测试断言中其隐藏状态 `.isHidden()` 必须为 `True`，同时操作系统中不可见。
2. **Given** 窗口已隐藏到系统托盘且状态标志位已正常重置，**When** 用户双击系统托盘上的 PhoneMic 图标或在托盘菜单中选择“显示主界面”，**Then** 主窗口应当还原为正常尺寸并弹回前台展示，成为当前活动焦点窗口。

---

### User Story 2 - 窗口关闭转换为隐藏 (Priority: P2)

用户在点击主窗口右上角“关闭 (X)”按钮时，程序不会真正退出，而是自动将其隐藏到系统托盘，避免因误操作关闭导致后台语音连接服务意外中断。

**Why this priority**: 许多工具型后台程序都采用关闭窗口即隐藏到托盘的设计，有助于提升程序的运行稳定性，防止误触。

**Independent Test**:
- **自动化单元测试（WSL Linux环境）**：运行 pytest 自动化测试。向主窗口分发关闭事件（`closeEvent`），拦截并断言该关闭事件已被忽略（`event.isAccepted()` 为 `False`），且窗口的隐藏状态为 `True`。
- **手动实机测试（Windows环境）**：启动程序，点击窗口右上角的“关闭 (X)”按钮，确认程序并没有退出进程，且手机端的语音数据仍可以正常传输。

**Acceptance Scenarios**:

1. **Given** 窗口处于显示状态，**When** 用户点击右上角“关闭”按钮（或窗口接收到关闭事件），**Then** 该关闭事件必须被拦截忽略（`event.ignore()`），窗口进入隐藏状态，程序主进程与后台接收服务不退出、不受影响。

---

### User Story 3 - 托盘菜单彻底退出 (Priority: P3)

用户需要彻底退出程序时，可以通过系统托盘图标的右键菜单进行操作。该退出操作不受窗口关闭拦截逻辑的影响，能够安全、彻底地结束进程。

**Why this priority**: 提供程序唯一的安全退出路径，使得退出逻辑真正生效。

**Independent Test**:
- **自动化单元测试（WSL Linux环境）**：运行 pytest 自动化测试。当主窗口的强制退出标志被触发为 `True` 时，分发关闭事件，断言该关闭事件已被接受（`event.isAccepted()` 为 `True`）。
- **手动实机测试（Windows环境）**：右键点击右下角系统托盘图标，选择“退出”，确认所有程序界面消失，且系统进程列表中没有 PhoneMic 相关的后台进程。

**Acceptance Scenarios**:

1. **Given** 强制退出标志已被托盘菜单退出操作设置为开启，**When** 窗口接收到关闭事件，**Then** 该关闭事件必须被接受（`event.accept()`），窗口销毁，关联的后台服务与主进程彻底终止，系统占用的网络端口和资源完全释放。

---

### Edge Cases

- **隐藏状态恢复后的尺寸异常**：当程序从最小化状态隐藏后，在托盘重新被双击唤醒时，窗口必须能够恢复为正常大小展示，而不能依然处于最小化状态。
- **快速切窗与双击托盘**：在窗口正在隐藏的过程中如果用户快速连续双击托盘，程序应当能正确处理窗口显示状态的切换，不应发生窗口冻结或异常崩溃。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 当主窗口触发最小化动作时，系统必须隐藏主窗口，使其彻底从操作系统的任务栏以及任务切换（如 Alt+Tab）视图中消失。
- **FR-002**: 系统在隐藏主窗口的同时，必须清除窗口的“最小化”状态属性，以便下次显示时以正常尺寸展现在桌面上。
- **FR-003**: 当主窗口触发关闭动作（如点击标题栏右上角的关闭按钮）时，系统必须将行为拦截并转换为隐藏主窗口，不能终止主程序进程。
- **FR-004**: 系统托盘的右键上下文菜单中必须提供一个明确的“退出”指令，点击该指令必须无条件地安全终止整个程序进程。
- **FR-005**: 主窗口处于隐藏状态（即最小化或关闭后）时，局域网连接和语音传输后台服务必须不受任何影响地持续运行。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 最小化窗口后，系统任务栏以及 `Alt+Tab` 切窗界面中该程序出现的比例必须为 0%（可通过真机手动验证）。
- **SC-002**: 从隐藏状态双击托盘图标或通过菜单选择展示窗口，主窗口在 0.5 秒内应当恢复显示并在最前端获得键盘/鼠标焦点。
- **SC-003**: 在测试驱动验证中，窗口关闭事件被 `ignore` 的比例应为 100%（当强制退出未开启时），且后台数据连接及模拟按键逻辑保持正常。
- **SC-004**: 从托盘菜单点击“退出”后，主进程及所有相关的网络端口监听在 2.0 秒内应当彻底结束并完成系统资源释放。

## Assumptions

- 用户的操作系统环境支持标准的系统托盘机制。
- 窗口最小化和关闭时隐藏的行为作为软件的内置默认逻辑，无需向用户提供是否启用的额外偏好设置开关。
- 本特性的改动不需要在用户界面上增加额外的多语言文本，直接沿用现有的“显示主界面”和“退出”等菜单项翻译。
