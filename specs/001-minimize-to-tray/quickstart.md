# Run & Validation Guide (Quickstart)

本指南指导开发者如何在 WSL 的本地测试环境下运行自动化单元测试，以及如何在 Windows 实机上进行部署和黑盒验证。

## 1. 自动化单元测试 (WSL 环境)

我们使用 `pytest` 作为测试执行引擎，在 WSL 的 Linux 虚拟环境中通过 Mock 的方式跑通窗口事件处理流程。

### 前置条件
- 已激活 poetry / uv 虚拟环境
- 安装了 `pytest` 和 `pytest-qt` 依赖包

### 单元测试执行命令
运行如下命令来专门执行新增加的托盘最小化/关闭拦截单元测试：

```bash
poetry run pytest tests/test_dashboard_tray.py -v
```

### 验证标准（断言）
自动化测试会覆盖：
1. **最小化隐藏断言**：模拟发送 WindowStateChange 事件，断言 `dashboard.isHidden() == True`，且 `dashboard.windowState() == Qt.WindowNoState`。
2. **关闭拦截断言**：直接调用 `dashboard.close()`，断言事件被 ignore（窗口未销毁），且 `dashboard.isHidden() == True`。
3. **托盘强制退出断言**：在设置 `_force_quit = True` 后，调用 `dashboard.close()`，断言关闭事件被接受，窗口顺利释放且 `QApplication` 的退出机制成功触发。

## 2. 手动实机功能确认 (Windows 实机环境)

本特性的系统级任务栏表现和 Alt+Tab 效果由于高度依赖操作系统的桌面管理器表现，需要在 Windows 真机下手动确认。

### 前置条件
- 使用 nuitka 或 PyInstaller 在 Windows 环境下打包，或者在 Windows 下使用 Python 直接启动程序。

### 操作确认步骤

1. **测试窗口最小化**：
   * 启动程序，在显示主界面的状态下，点击标题栏右上角的“最小化”按钮。
   * **预期结果**：主窗口从屏幕和任务栏彻底消失。使用 `Alt+Tab` 切窗界面无法找到该程序。右下角托盘中的图标仍正常存在。
2. **测试隐藏还原**：
   * 在窗口隐藏的状态下，双击托盘图标，或者右键托盘图标选择“显示主界面”。
   * **预期结果**：主窗口立刻弹回，并且其窗口尺寸大小恢复正常（不是最小化状态），主窗口获取当前操作焦点。
3. **测试防误触关闭**：
   * 在主窗口显示状态下，点击右上角的“关闭 (X)”按钮。
   * **预期结果**：窗口从桌面和任务栏消失（同最小化行为），程序进程继续在后台运行，手机扫码依然可连接且语音输入能无缝打字。
4. **测试彻底退出**：
   * 右键点击系统托盘图标，选择“退出”。
   * **预期结果**：程序窗口全部销毁，网络监听端口释放，进程安全结束。
